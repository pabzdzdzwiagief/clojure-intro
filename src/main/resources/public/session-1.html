That was the story back then, but probably still holds. Clojure started off as a language for dealing with concurrency. To remain sane when writing concurrent programs maintaining as much immutability as possible the key. And FP makes sense for dealing with immutable data.
Might also mention that it is meant to be stable, and to have that without suffering from stagnation you need extensibility (macros).
Syntax - against popular conception that all programming languages are alike, Clojure might at first feel really alien to someone without prior exposure to any LISP. Apart from syntax what really draws a line between Java and Clojure is taking sceptical stance towards Object Orientation and instead emphasizing manipulation of plain immutable data. Note that despite emphasis on pure data, under the hood everything is done with Java objects.
